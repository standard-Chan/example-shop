# example-shop

깔끔한 코드와 유지보수 가능한 구조를 고민하며 만들어가는 `Spring Boot` 프로젝트입니다.
단순 기능 구현이 아닌, 읽기 쉽고 일관된 비즈니스 로직 작성을 목표로 합니다.

---

## 목표

* 명확하게 읽히는 비즈니스 로직 작성
* 객체지향적인 코드 (규칙을 domain에 집중)
* 역할이 분리된 계층 구조 설계
* 개인적인 "좋은 코드 기준" 정리

## 시도
* service 단의 복잡성을 줄이고, 각 주요 로직을 도메인으로 옮긴다.
* 이를 통해 application은 요청의 흐름을 쉽게 파악할 수 있게 한다
* domain 모듈에서 핵심 비즈니스 로직을 처리한다.
* application은 domain의 기능을 조합하여 비즈니스 로직을 구현한다.

---

## 프로젝트 방향

* Controller는 요청/응답 처리에만 집중
* Service는 흐름 제어 역할만 수행
* 비즈니스 로직은 Domain에 위치
* DTO ↔ Entity 변환 명확히 분리
* 테스트 가능한 구조 유지

---

## 주요 기능

* 상품 등록 / 조회
* 주문 생성 / 취소
* 결제 처리
