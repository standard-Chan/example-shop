# example-shop

깔끔한 코드와 유지보수 가능한 구조를 고민하며 만들어가는 `Spring Boot` 프로젝트입니다.
단순 기능 구현이 아닌, 읽기 쉽고 일관된 비즈니스 로직 작성을 목표로 합니다.

---

## 목표

* 명확하게 읽히는 비즈니스 로직 작성
* 역할이 분리된 계층 구조 설계
* 변경에 유연한 코드 구조 만들기
* 개인적인 "좋은 코드 기준" 정리

---

## 프로젝트 방향

* Controller는 요청/응답 처리에만 집중
* Service는 흐름 제어 역할만 수행
* 비즈니스 로직은 Domain에 위치
* DTO ↔ Entity 변환 명확히 분리
* 테스트 가능한 구조 유지

---

## 주요 기능

* 상품 등록 / 조회
* 주문 생성 / 취소
* 결제 처리
